8-Step AI Pipeline

From requirement to production-ready code, fully automated.

1

Branch

Automatically creates feature branches when on main/master with AI-powered naming based on your requirement.

2

Implement

Takes your requirement and asks Claude to implement it following your project's conventions and patterns.

3

Simplify

Refines the implementation for clarity, consistency, and maintainability while preserving all functionality.

4

Review

Reviews code for bugs, security issues, performance problems, and maintainability concerns.

5

SOLID Check

Ensures compliance with SOLID principles and clean code practices for long-term maintainability.

6

Test

Runs existing tests, adds new tests for changed code, and verifies coverage meets standards.

7

Commit

Creates well-formatted commits with conventional commit format and detailed explanations.

8

Changelog

Updates CHANGELOG.md following the Keep a Changelog format automatically.

Usage

Simple CLI with powerful options.

Examples

# Basic usage
sf "add user authentication with JWT"

# Adaptive mode - AI skips irrelevant steps
sf --adaptive "fix typo in README"

# Custom review iterations
sf --reviews 3 "refactor database layer"

# Dry run to preview commands
sf --dry-run "add dark mode toggle"

# Skip tests and push
sf --skip-tests --skip-push "update README"

# Log output to file
sf --log pipeline.log "implement caching"

Options

Option Description
-a, --adaptive AI skips irrelevant steps based on requirement
-d, --dry-run Print commands without executing
-r, --reviews <n> Number of review iterations (default: 2)
--skip-tests Skip the testing step
--skip-push Commit but don't push to remote
--skip-branch-management Skip automatic branch creation
--log <file> Log output to file

Get Started
